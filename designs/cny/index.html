Project Management Methodology

We use Agile methodology as a project management approach that emphasizes flexibility, collaboration, and customer focus. It allows us to be more responsive to change and to deliver short phases of work with frequent testing, reassessment, and adaptation throughout. We also use Scrum framework that provides a structure for managing and completing complex projects. It includes roles such as Scrum Master, Product Owner, and Development Team and limits work-in-progress to improve flow and reduce delays. Work is split into short cycles known as “sprints”, teams are led by a Scrum Master for the duration of the sprint, after which they review their performance in a “sprint retrospective” and make any necessary changes before starting the next sprint.

Tools for project management: Azure DevOps Services (TFS), Jira Work Management, Confluence, Trello, etc.

Native and Hybrid
Mobile Technologies:

We are well-versed in creating applications that run on mobile devices such as smartphones and tablets. We cover different platforms such as iOS and Android, and responsive designs that adapt to different screen sizes and resolutions:

Databases:

We utilize various types of databases in order to achieve optimal performance, scalability, and flexibility for different types of data and use cases.

We use several types of databases. Some of the most common types that we use are:

  • Relational databases: Table-based structured format like MySQL, Microsoft SQL Server, PostgreSQL, Oracle
  • Document-oriented databases: Semi-structured data format, JSON, like MongoDB or Elasticsearch.
  • Column-family databases: Data is stored in a column-based format, like Cassandra.
  • Key-value databases: These databases store data as key-value pairs, like Redis and DynamoDB
  • In-memory databases: These databases store data in memory, rather than on disk, for faster performance like MemSQL and SAP HANA.

Testing:

Some of the most popular testing frameworks we use:

  • Jest: developed and maintained by Facebook, it is widely used for unit and integration testing.
  • Mocha: a flexible and feature-rich testing framework that supports both browser and Node.js environments.
  • Jasmine: a behavior-driven development (BDD) framework for testing JavaScript code.
  • Cypress: an end-to-end testing framework that aims to make browser automation more approachable and less flaky.
  • Puppeteer: a Node.js library that provides a high-level API to control Chrome or Chromium over the DevTools Protocol.
  • Karma: A test runner for browser-based JavaScript code. Allows the application script to be executed on real browsers and devices like phones and tablets.

Hosting and
Analytics:

We use these comprehensive, scalable, and secure cloud computing platforms:

Cloud hosting is highly scalable and can be used for a variety of applications, like storage and data processing:

  • Microsoft Azure, Amazon Cloud, Google Cloud

These are several types of hosting we use:

  • Shared hosting: This is the most affordable and basic type of hosting, where multiple websites share the same server and resources.
  • VPS (Virtual Private Server) hosting: Virtual machine that runs on a physical server, allowing users to have their own dedicated resources.
  • Dedicated hosting: This is the most expensive type of hosting, where a user rents an entire physical server for their exclusive use.
  • Managed hosting: This type of hosting is where a provider manages the server on behalf of the user, providing software updates, backups, and security.

We use the following Analytics tools to gain insights:

  • New Relic, Google Analytics, Our custom solutions

Build and
Automation:

We use the following Build automation tools that automate the process of compiling, packaging, and deploying software applications:

  • Azure DevOps: (formerly known as Visual Studio Team Services) - a set of development tools for software teams, provided by Microsoft.
  • MSBuild: (Microsoft Build Engine) is a build tool developed by Microsoft that is used to build Visual Studio projects and solutions.
  • Azure Pipelines: is a cloud-based continuous integration and continuous delivery (CI/CD) service provided by Microsoft as part of the Azure DevOps suite.
  • TeamCity: a continuous integration and build management tool developed by JetBrains. It provides a web-based user interface, a built-in VCS, and a lot of integrations with other tools.
  • Jenkins: an open-source automation server that provides hundreds of plugins to support building, deploying, and automating any project.
  • PowerShell: is a powerful automation tool and scripting language developed by Microsoft. It is built on top of the .NET Framework and provides a command-line interface (CLI) and a scripting environment for Windows.
